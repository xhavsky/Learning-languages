Jak pracujemy (żeby się nie zderzyć)

  1. Jeden projekt — ten sam folder / GitHub main. Nie ma drugiego „WIP”.
  2. Jedna osoba naraz przy kodzie — jeśli tu świeci pasek („asystent” albo „tata lokalnie”), druga osoba czeka.
  3. Jedna prośba naraz — Anielka czeka na koniec (pasek postępu), dopiero potem następna zmiana.
  4. Mówcie głośno — w czacie: „biorę słówka”, „biorę UI”, „robię release”.
  5. Commit przed paczką — Release tylko gdy zmiany są zapisane w Gicie (inaczej przycisk odmówi).
  6. Nie dublujcie — nie róbcie tej samej rzeczy lokalnie i przez portal w tym samym czasie.
